### Lazily.AI: A Technical Overview and Strategic Future

**Project Overview**
Lazily.AI is a specialized Software-as-a-Service (SaaS) platform designed to **automate the generation of Texas Real Estate Commission (TREC) 1-4 Family Residential Contracts**. It aims to eliminate the tedious, time-consuming, and error-prone task of manually filling out these contracts for real estate professionals. The platform allows users to upload a CSV file with property and owner data to instantly generate compliant, error-free PDF contracts. This transforms a multi-hour administrative burden into a task that takes mere seconds, providing a distinct competitive advantage.

**Core Workflow**
The core workflow for contract generation involves several steps:
1.  **Upload CSV**: The user uploads a CSV file, typically using the provided template found at `public/templates/lazily-ai-template.csv`.
2.  **Parse & Validate**: The backend parses the CSV using Papa Parse and **validates each row against a comprehensive Zod schema** (located in `lib/contracts/validation.ts`).
3.  **Credit Check**: The system verifies that the user's team has enough contract credits for the number of rows uploaded.
4.  **Save Data**: Valid contract data is stored in the PostgreSQL database in the `contracts` table, and the appropriate number of credits are deducted in an atomic transaction.
5.  **Generate PDF (On-Demand)**: When the user requests a download, the system retrieves the saved data and generates the PDF using the `pdf-lib` library and the TREC template (`lib/templates/TREC-20-18-automated-v1.pdf`). The generated PDF is **flattened to be non-editable**.

**Tech Stack**
Lazily.AI leverages a modern, serverless-friendly technology stack focused on rapid iteration, scalability, and minimal operational cost.

*   **Framework**: Next.js 14 (App Router). It utilizes **Next.js Server Actions for most backend operations** initiated from the client, minimizing client-side JavaScript and co-locating data mutations.
*   **Deployment & Hosting**: Vercel.
*   **Database**: Postgres (specifically Neon Serverless Postgres).
*   **ORM**: Drizzle ORM. All database queries are made using Drizzle ORM, with the schema defined in `lib/db/schema.ts`.
*   **Payments**: Stripe. Stripe is used for managing both recurring subscriptions (platform access) and one-time credit purchases (contract generation). Stripe webhooks handle events to update subscription statuses and credit balances.
*   **UI Components**: shadcn/ui & Tailwind CSS.
*   **Validation**: Zod, used as the single source of truth for contract data structure (`lib/contracts/validation.ts`).
*   **CSV Parsing**: Papa Parse.
*   **PDF Manipulation**: `pdf-lib`.
*   **Authentication**: JWT/cookies using `jose`. Passwords are securely hashed using `bcryptjs`.

**Strategic Upgrades and Future Roadmap**
Lazily.AI has a clear roadmap for future growth and feature expansion, aiming to become a more comprehensive tool for Texas real estate professionals.

*   **E-Signature Integration**: The platform plans to integrate with an e-signature service to allow users to send contracts for signature directly. **Dropbox Sign (formerly HelloSign) is the recommended partner** due to its more accessible pricing model and developer-friendly API, which aligns with Lazily.AI's focus on speed and simplicity. This integration will involve a server action to initiate signature requests, attaching the generated PDF and signer information, and a webhook handler to update contract statuses upon completion.
*   **CRM Integration ("Bots")**: To further eliminate manual data entry, Lazily.AI plans to build "bots" (serverless functions) that can pre-fill contract data directly from popular real estate CRMs. **Follow Up Boss and kvCORE are high-priority targets** due to their accessible REST APIs. The architecture will use a **pluggable, serverless approach with an Adapter Pattern** to standardize CRM data into a `CrmContact` interface, ensuring modularity and extensibility. User CRM API keys will be securely encrypted before storage.
*   **Advanced Analytics Dashboard**: An analytics dashboard will be added to provide users with insights into their operations, such as contract generation volume and offer acceptance rates. **Tremor is the superior choice for data visualization** due to its compatibility with the existing ShadCN/UI and Tailwind CSS design system, accelerating development and ensuring visual consistency. Key metrics will include offer acceptance rate, contracts generated vs. offers accepted monthly, total contract volume over time, and credit usage breakdown.
*   **Hybrid Input System**: To cater to different user needs (bulk vs. single entry), a multi-modal input system is planned.
    *   **Phase 1: Single Contract Form**: A web form for generating a handful of contracts, featuring real-time validation and tooltips.
    *   **Phase 2: Intelligent Import Wizard**: An improved CSV upload experience with flexible column mapping, data preview, and in-browser correction of errors.
    *   **Phase 3: In-App Spreadsheet**: A power user feature allowing direct data entry and copy-pasting from Excel into an interactive, validating table.
*   **Expanded Form Library**: Beyond the TREC 1-4 contract, the platform may expand to include other common Texas real estate forms.
*   **API Access**: Lazily.AI could offer a public API to allow other PropTech companies or large brokerages to integrate its document generation engine directly into their custom workflows.
